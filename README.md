# E-Commerce Revenue & Retail Performance Analysis

## 📌 Project Overview
This project delivers an end-to-end data analytics solution designed to optimize a retail business's catalog performance and regional sales strategy. Using a modern hybrid data architecture, I built a structured SQL relational database to handle data wrangling and multi-table aggregations, serving as the backend data pipeline. For the frontend presentation, I engineered an interactive, stakeholder-ready executive dashboard in Microsoft Excel via Power Query.

## 🛠️ Tech Stack & Tools
- **Database Engine:** SQL / PostgreSQL (Data modeling, relational inner joins, business logic filtering)
- **Reporting Interface:** Microsoft Excel (Power Query data injection, Pivot Tables, UI/UX Dashboard Design)

## 🗄️ Relational Data Schema & Logic
The backend utilizes three primary tables structured with strict relational integrity (`customers`, `products`, and `orders`). To ensure strict financial accuracy, the analytics engine filters out operational friction by dropping cancelled orders, isolating true realized revenue for leadership decision-making.

## 📊 Core Business Insights & Impact
- **Computing Dominance:** The Computing category serves as the primary engine of value, generating the highest total revenue share due to premium unit price points.
- **Regional Variations:** Distinct sales spikes appear across key demographic regions, allowing marketing teams to optimize geographic ad spend.
- **Transactional Strength:** While certain categories hold lower volume, their high Average Order Value (AOV) indicates extreme high-margin performance efficiency per checkout event.
